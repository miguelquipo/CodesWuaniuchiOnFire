Programación en Arduino

La programación en Arduino se refiere a la creación de código de software para controlar dispositivos y sistemas utilizando placas de desarrollo de hardware Arduino. Arduino es una plataforma de código abierto ampliamente utilizada que consta de una placa de microcontrolador y un entorno de desarrollo integrado (IDE) que simplifica la programación de microcontroladores. La programación en Arduino se utiliza en una variedad de proyectos, desde proyectos de robótica y automatización hasta sistemas de control de luces, sensores y dispositivos interactivos.


  • > Microcontroladores Arduino: Las placas Arduino están equipadas con microcontroladores que varían según el modelo. Los microcontroladores más comunes son los de la familia AVR de Atmel (ahora propiedad de Microchip) y los microcontroladores basados en ARM.

  • > Entorno de Desarrollo Arduino (Arduino IDE): Es un software que proporciona un conjunto de herramientas para escribir, compilar y cargar código en una placa Arduino. El IDE también incluye una biblioteca estándar de funciones que facilitan la programación de tareas comunes, como la lectura de sensores y el control de actuadores.

  • > Lenguaje de Programación: Arduino utiliza un lenguaje de programación basado en C/C++. Aunque la sintaxis puede parecer intimidante para los principiantes, Arduino simplifica muchas tareas de bajo nivel y ofrece una serie de bibliotecas predefinidas que facilitan la programación.

  • > Librerías: Arduino cuenta con una amplia variedad de bibliotecas disponibles que simplifican la programación de dispositivos y sensores específicos, como pantallas LCD, sensores de temperatura, módulos WiFi, entre otros.

  • > Estructura de un Programa: Un programa típico de Arduino consta de dos funciones principales: setup() y loop(). La función setup() se ejecuta una vez al principio y se utiliza para la inicialización de variables y configuración de pines. La función loop() se ejecuta de forma continua y contiene el código principal del programa.

  • > Hardware: Arduino permite interactuar con el mundo físico a través de pines de entrada/salida (GPIO) que pueden utilizarse para conectar sensores, actuadores y otros dispositivos.

Programación de Aplicación Movil



La programación de aplicaciones móviles en Android Studio es un proceso fundamental para desarrollar aplicaciones para dispositivos Android. Android Studio es el entorno de desarrollo integrado (IDE) oficial de Android proporcionado por Google, y es ampliamente utilizado por desarrolladores para crear aplicaciones móviles Android.

Android Studio es una herramienta poderosa para el desarrollo de aplicaciones móviles Android, y ofrece una amplia gama de recursos y documentación para ayudarte a crear aplicaciones de alta calidad y funcionalidad. Además, el aprendizaje de Java o Kotlin es fundamental para el desarrollo de aplicaciones en Android Studio, así como familiarizarse con los conceptos de programación móvil.

Pruebas de servo


Conexiones Electricas

• Conecta el cable de alimentación del servomotor (rojo) al pin de 5V en el Arduino.
• Conecta el cable de tierra del servomotor (negro o marrón) al pin GND en el Arduino.
• Conecta el cable de señal del servomotor (blanco, amarillo o naranja) al pin digital del Arduino, por ejemplo, al pin 9.

Codigo de lectura

• Abre el software Arduino IDE en tu computadora.
• Ve a Archivo > Ejemplos > Servo > Sweep para abrir un ejemplo básico que hace que el servomotor se mueva de un extremo a otro.
• El código de ejemplo se verá más o menos así: 

     #include 
        Servo myservo; 
        void setup() { 
            myservo.attach(9); 
        } 
        void loop() {
            for (int pos = 0; pos <= 180; pos += 1) {
            myservo.write(pos);
            delay(15);
            }
            for (int pos = 180; pos >= 0; pos -= 1) {
            myservo.write(pos); 
            delay(15); 
            }
        }

Pruebas de Relé


Conexiones Electricas

• Conecta el cable de control del relé al Arduino. Generalmente, esto se hace conectando un pin del Arduino (por ejemplo, el pin 7) al pin de control (IN) del relé.
• Conecta la fuente de energía a la entrada de alimentación del relé. Asegúrate de que la tensión de alimentación sea compatible con las especificaciones del relé (generalmente 5V o 12V, dependiendo de la variante).
• Conecta el pin COM (común) del relé a una fuente de alimentación externa (como una batería) para alimentar el dispositivo que deseas controlar.
• Conecta uno de los terminales NO (normalmente abierto) del relé al polo positivo del dispositivo que deseas controlar.
• Conecta el otro terminal NO del relé al polo negativo del dispositivo que deseas controlar.

Codigo de lectura

• Abre el software Arduino IDE en tu computadora.
• Utiliza el siguiente código de ejemplo para encender y apagar el relé: 

        int relayPin = 7;
        void setup() { 
        pinMode(relayPin, OUTPUT); 
        } 
        void loop() { 
        digitalWrite(relayPin, HIGH); // Encender el relé 
        delay(1000); // Esperar 1 segundo 
        digitalWrite(relayPin, LOW); // Apagar el relé 
        delay(1000); // Esperar 1 segundo 
        }

Pruebas del Driver


Conexiones Electricas

• Conecta el pin de habilitación (ENA) del primer motor al pin 9 en el Arduino.
• Conecta el pin de control (IN1) del primer motor al pin 10 en el Arduino.
• Conecta el pin de habilitación (ENB) del segundo motor al pin 5 en el Arduino.
• Conecta el pin de control (IN3) del segundo motor al pin 6 en el Arduino.
• Conecta los pines de salida (OUT1 y OUT2) del primer motor al terminal positivo y negativo, respectivamente.
• Conecta los pines de salida (OUT3 y OUT4) del segundo motor al terminal positivo y negativo, respectivamente.
• Conecta el terminal negativo de ambos motores al terminal negativo de la fuente de energía y al pin de tierra (GND) del L298N.
• Conecta la fuente de energía a los terminales de alimentación (12V) del L298N.
• Conecta los pines de tierra (GND) de la fuente de energía y del Arduino entre sí.

Codigo de lectura

• Abre el software Arduino IDE en tu computadora.
• Utiliza el siguiente código de ejemplo para controlar ambos motores: 

        int enA = 9; 
        int in1 = 10; 
        int enB = 5; 
        int in3 = 6; 
        void setup() { 
        pinMode(enA, OUTPUT); 
        pinMode(in1, OUTPUT); 
        pinMode(enB, OUTPUT);
        pinMode(in3, OUTPUT);
        } 
        void loop() { 
        digitalWrite(in1, HIGH); 
        analogWrite(enA, 255);
        digitalWrite(in3, HIGH); 
        analogWrite(enB, 255); 
        delay(2000); // Detiene ambos motores 
        analogWrite(enA, 0);
        analogWrite(enB, 0);
        delay(1000);
        digitalWrite(in1, LOW); 
        analogWrite(enA, 255); 
        digitalWrite(in3, LOW); 
        analogWrite(enB, 255); 
    }

Pruebas del Sensor de Ultrasonido


Conexiones Electricas

• Conecta el pin VCC del sensor HC-SR04 a 5V en el Arduino.
• Conecta el pin GND del sensor HC-SR04 a GND en el Arduino.
• Conecta el pin TRIG del sensor HC-SR04 a un pin digital en el Arduino, por ejemplo, el pin 9.
• Conecta el pin ECHO del sensor HC-SR04 a otro pin digital en el Arduino, por ejemplo, el pin 10.

Codigo de lectura

• Abre el software Arduino IDE en tu computadora.
• Utiliza el siguiente código de ejemplo para medir la distancia y mostrarla en el monitor serial: 

        #define trigPin 9 
        #define echoPin 10 
        void setup() { 
        Serial.begin(9600); 
        pinMode(trigPin, OUTPUT); 
        pinMode(echoPin, INPUT); 
        }
        void loop() { 
        long duration;
        int distance; 
        digitalWrite(trigPin, LOW); 
        delayMicroseconds(2); 
        digitalWrite(trigPin, HIGH); 
        delayMicroseconds(10);
        digitalWrite(trigPin, LOW); 
        duration = pulseIn(echoPin, HIGH);
        distance = (duration / 2) / 29.1; 
        Serial.print("Distancia: "); 
        Serial.print(distance); 
        Serial.println(" cm");
        delay(1000); 
        }

Pruebas Mix


Conexiones Electricas

• Conecta el pin SDA del MLX90614 al pin SDA (A4 en Arduino Uno) en el Arduino.
• Conecta el pin SCL del MLX90614 al pin SCL (A5 en Arduino Uno) en el Arduino.
• Conecta el pin VCC del MLX90614 a la alimentación de 3.3V o 5V en el Arduino.
• Conecta el pin GND del MLX90614 al pin GND en el Arduino.

Codigo de lectura

• Abre el software Arduino IDE en tu computadora.
• Asegúrate de que tengas la librería adecuada instalada. En el Arduino IDE, ve a Sketch > Incluir Biblioteca > Administrar Bibliotecas, busca "Adafruit MLX90614" e instálala si no está instalada.
• Utiliza el siguiente código de ejemplo para leer la temperatura del sensor y mostrarla en el monitor serial: 

        #include  
        #include 
        Adafruit_MLX90614 mlx = Adafruit_MLX90614();
        void setup() {
        Serial.begin(9600); 
        mlx.begin(); 
        } 
        void loop() {
            float tempAmbient = mlx.readAmbientTempC(); 
            float tempObject = mlx.readObjectTempC(); 
            Serial.print("Temperatura ambiente: "); 
            Serial.print(tempAmbient); 
            Serial.println(" °C"); 
            Serial.print("Temperatura del objeto: "); 
            Serial.print(tempObject);
            Serial.println(" °C"); 
            delay(1000); 
        }

Proceso de armado

Una vez culminado este proceso y verificando que los componentes estén en buen estado, comenzamos con la conexión.

Ubicamos nuestra base en donde vamos a ubicar nuestros implementos, en nuestro caso una tabla que tiene el tamaño adecuado.

Comenzamos con la ubicación de los motores que llevaran las llantas, en el lado de abajo que destinemos. Al igual que una rueda giratoria en la parte media delantera.

En la parte contraria a donde pegamos las llantas pegamos el resto de componentes como lo son el Arduino, en nuestro caso Arduino Mega, Driver L298N, Protoboard, Relé, Servos, mlx, y en la parte de las llantas los tres sensores de llama ky-026.

Ahora en parte superior, conjunto a un servo aplicamos nuestro sensor de ultra sonido que dará señales de ultra sonido para medir la distancia y evadir obstáculos.

Ahora comenzamos con la conexión de los componentes, incoando con los cables de los motores, los cuales soldamos previamente y los extendemos con cables de Protoboard macho – macho y macho – hembra (antes de esto es preferible verificar la polaridad de los motores para que no haya mucho conflicto en tanto al código).

Lo conectamos a dos canales del Protoboard para consiguiente establecer las conexiones con los 4 “OUT” que tiene nuestro driver.

Ahora con el driver, identificamos los pines ENA, ENB, IN1 ,IN2,IN3 Y IN4, en el caso de los ENA Y B generamos una unión en común para sacarlo como un solo canal para conectarlo al Arduino.

En el Arduino identificamos los Pines 7,6,5,4,3,2, donde 7 y 2 corresponden a los PWM que darán potencia a nuestros motores, y los demás harán el trabajo de orientación digital.

Ya instalado el driver, continuamos con los sensores, tanto de llama, infrarrojo y ultrasonido. En el Protoboard sacamos 5V a cualquiera de los buses de energía, dentro de los canales de conexión designamos que pines a ocupar el positivo (5 V) y el negativo (GND) del Arduino respectivamente.

En el caso de los sensores de llama “ky-026” disponemos de los piense de alimentación G (GND) Y + (POSITIVO), y en este caso de los tres sensores que ya habíamos colocado tenemos un pin llamado “D0” que recibirá datos booleanos que para realizar el funcionamiento de la búsqueda de llamas o temperaturas elevadas de las cuales hablaremos en la parte de software, estos van conectados distintivamente en los pines 8,9,10 del Arduino.

En el caso de nuestro sensor infrarrojo hacemos las conexiones de alimentación donde “VIN” es positivo y “GND” negativo, en el caso del sensor, y los puertos SCL, SDA son puertos analógicos que en el caso del Arduino Mega son pines ya designado y no es necesario conectarlos en pines analógicos como en Arduino UNO, en este caso lo conectamos según corresponde.

Conexión Relé: este al igual que los sensores puede y debe ser alimentado a tras de sus pines “VCC” positivo y “GND” negativo, estos le darán la alimentación para que pueda activarse según el sistema de Arduino, donde el puerto “IN” será e que reciba la señal para activar el mecanismo de la bomba de agua en el caso de este proyecto, en el ejercicio nosotros le asignamos el Pin digital número 11 en Arduino Mega.

En el caso de los servos también necesitamos alimentarlos según sus salidas, estos no tienen nombres, así que nos guamos por sus colores, donde Rojo es positivo, Marrón Negativo, Naranja es la señal de la instrucción que recibirá, asignamos los pines 32 y 33 en el Arduino

Conexión sensor Ultra Sonido, en este caso tenemos la misma situación, alimentación del mismo, pero con salidas digitales “ECHO” y “TRIG”, que en el caso del proyecto asignamos los pines 22, y 24.

Ahora debemos alimentar nuestro circuito, en el caso de este proyecto utilizamos una fuente de energía de pilas de 3.7 V cada una que darán energía a las llantas, o más específicamente al driver, donde a través de otro bus de energía en el Protoboard alimentaremos el Driver en la conexión de 12 V y GND. En cambio, con otra fuente aparte de 8 V (Powerbank), alimentamos al Arduino Mega.
